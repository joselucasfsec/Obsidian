# 1 - Recon
 **Scope e Ouf of scope**
Identificar domínios e endereços IPs que estão no escopo dos testes.
**Subdomain discovery**
Processo para identificar subdomínios dentro de domínios, você pode utilizar várias técnicas para este processo.
**Content Discovery**
Processo para identificar informações e dados que podem ser interessantes dentro do processo de identificação e análise de vulnerabilidades.
**Application Discovery**
Este é um ponto que é utilizado para identificar se o seu alvo está utilizando alguma ferramenta ou software de terceiro, esta etapa pode ser bem interessante, onde sempre há possibilidade de existir vulnerabilidade nestes softwares terceiros e o seu alvo ainda não ter atualizado o software.
# 2 - Identificação e Análise
**Filtro e validação**
Identificar se o seu alvo está rodando o servidor HTTP, por exemplo. Poderíamos utilizar alguma ferramenta para auxiliar nos testes como httpx.
**Vulnerability assessment**
Processo para identificar vulnerabilidades nos ativos, quando estamos falando de Bug Bounty normalmente este é um processo manual.
**Validação e PoC**
Falando do cenário de Bug Bounty o processo na maioria das vezes se encerra com a validação da vulnerabilidade e sua existência com uma PoC.
